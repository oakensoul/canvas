# Security Policy

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in **oakensoul-canvas**, please report it responsibly.

### Preferred: GitHub Security Advisories

Please use [GitHub Security Advisories](https://github.com/oakensoul/canvas/security/advisories/new) to report vulnerabilities. This allows us to discuss and resolve the issue privately before public disclosure.

### Fallback: Email

If you are unable to use GitHub Security Advisories, you may email us at:

**github@oakensoul.com**

Please include as much detail as possible:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgement**: Within **72 hours** of receiving your report.
- **Assessment**: Within **7 days**, we will provide an initial assessment of the vulnerability, including severity and planned remediation timeline.

## Scope

This security policy covers:

- The `canvas` CLI tool
- Direct dependencies of the `canvas` package

Out of scope:

- Transitive dependencies (report these to the upstream project)
- Infrastructure or hosting (this is a local CLI tool)

## Contact

- **Email**: github@oakensoul.com
- **GitHub Security Advisories**: [Report a vulnerability](https://github.com/oakensoul/canvas/security/advisories/new)
